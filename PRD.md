## 1. 项目背景

### 1.1 现状与痛点

Kubernetes 已成为容器编排的事实标准，但现有的管理工具（如 Kubernetes Dashboard、Lens、Rancher）存在以下问题：

| 痛点 | 说明 |
|------|------|
| 部署重 | Dashboard 等工具需要独立部署，占用集群资源 |
| 依赖多 | 需要 Node.js、Python 等运行时环境 |
| 访问受限 | 需要通过端口转发或 Ingress 暴露，增加安全风险 |
| 场景不匹配 | 在家庭实验室、边缘计算等轻量场景中，重型管理工具过于冗余 |

### 1.2 机会

OpenWrt/iStoreOS 路由器是家庭网络和边缘场景中的常驻设备，具备以下优势：

- 7x24 小时在线，天然适合做管理入口
- 自带 LuCI Web 管理界面，无需额外部署
- 网络可达性好，可直接访问局域网内的 K8s API Server
- 手机/电脑均可通过浏览器访问

### 1.3 项目愿景

**让路由器成为轻量级 Kubernetes 控制中心** -- 无需额外部署管理面板，在路由器的 LuCI 界面中即可完成 K8s 集群的日常管理操作。

## 3. 使用场景

### 场景 1: 家庭 K8s 实验室日常管理

> 小张在家中软路由上搭建了 K3s 集群，运行着个人博客、Git 仓库等服务。他希望在手机上随时查看 Pod 运行状态，重启异常部署，查看容器日志。

**需求**: 集群总览、Pod 管理、Deployment 重启、日志查看

### 场景 2: 边缘节点监控

> 某公司在多个边缘站点部署了 K8s 节点，每个站点有一台 OpenWrt 路由器。运维人员希望通过路由器 Web 界面监控节点 CPU/内存使用率，及时发现异常。

**需求**: 节点状态监控、资源使用率展示、事件查看

### 场景 3: 快速 YAML 调试

> 开发者在调试 K8s 资源 YAML 配置时，希望快速查看和修改资源定义并立即应用，而不需要切换到 kubectl 命令行。

**需求**: YAML 查看/编辑/应用、资源详情查看

### 场景 4: 多集群切换管理

> 管理员同时维护开发、测试、生产多个 K8s 集群，每个集群有独立的 kubeconfig 文件，需要快速切换查看不同集群的状态。

**需求**: 多 kubeconfig 切换、历史记录

## 6. 系统约束

### 6.1 运行环境约束

```
目标平台: OpenWrt / iStoreOS 路由器
运行时: LuCI Web Framework (uhttpd/rpcd)
后端语言: Lua 5.1+
前端语言: JavaScript (ES5+)
依赖工具: kubectl, curl, jq
```

### 6.2 网络约束

- 路由器需能网络访问 K8s API Server
- kubeconfig 文件需预先部署到路由器上（默认路径 `/root/.kube/config`）
- kubectl 需预先安装在路由器上

### 6.3 资源约束

- 路由器 RAM 有限（256MB ~ 1GB），插件需极度轻量
- 存储空间有限，安装包需尽可能小
- CPU 性能有限，避免复杂计算
